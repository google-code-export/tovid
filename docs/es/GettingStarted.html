Este capítulo explica cómo conseguir que tovid funcione en tu ordenador. Si ya tienes tovid en marcha, puedes saltar directamente a Usando el interfaz gráfico (GUI) o Uso de las herramientas de la línea de comandos.

1. Preliminaries

1.1. Requerimientos

No hay requerimientos mínimos de sistema para ejecutar tovid. Si tu máquina puede arrancar una instalación Linux normal, probablemente ya tienes suficiente velocidad de CPU para procesar y reproducir vídeo.

Si vas a trabajar conficheros de vídeo, querrás tener espacio en disco de sobras; el proceso de autoría de un DVD puede consumir 11GB de tu disco, y si trabajas en varios proyectos de DVD a la vez, los requerimientos de espacio se multiplican rápidamente. Como mínimo, deberías disponer de unos 20GB que puedas dedicar a la autoría de DVDes. Si sólo planeas crear (S)VCDs, puedes ir tirando con sólo 6-8GB.

La familia tovid se desarrolla y comprueba principalmente en Linux. Algunos usuarios han informado de que funciona exitosamente en FreeBSD y Cygwin con sólo modificaciones menores. Parte de los componentes puede funcionar en otros sistemas operativos parecidos a UNIX, pero tal posibilidad no ha sido verificada de momento. Actualmente no hay planes para desarrollar una versión en Windows o Mac OS, pero si estás interesado en emprender el trabajo, ¡por favor contacta con los desarrolladores de tovid!

Se requiren varios paquetes de programas para poder usar los componente de tovid (mira Dependencias). Casi todos los componentes, al estar escritos en el lenguaje de script bash, requieren bash, y también requieren los comandos habituales en UNIX, como grep, sed, cat, y awk. Si tienes Linux, todos estos comandos están disponibles.

1.2. Dependencias

tovid ha acumulado su buen montoncillo de dependencias a lo largo de su desarrollo. Aquí está la lista completa de dependencias, y hasta qué punto son requeridos:

Python

Un intérprete que permite ejecutar programas escritos en Python. Python es cada vez más habitual, así que es bueno tener un intérprete de Python instalado. Es necesario sólo para el interfaz gráfico de tovid.

wxPython

Da soporte a GUIs escritos en Python usando la API wxWidgets. También solo para el GUI.

mplayer

Probablemente el reproductor multimedia más robusto y maduro para Linux. Casi todos los componenete de tovid dependen fuertemente de mplayer para reproducir y convertir vídeo, así que su instalación es necesaria. Aunque seguro que querrás usar mplayer de todos modos, si piensas trabajar con vídeo en Linux.

mjpegtools

Herramientas de línea de comando para codificar y trabajar con vídeo, particularmente MPEG. Requerido por la mayor parte de los componentes de tovid, y lo necesitarás pra cualquier cosa útil que quieras hacer con tovid.

ffmpeg

Herramientas para grabar, convertir, y enviar por la red audio y vídeo. Usados por varios componentes, y requeridos si quieres crear DVDs en formato NTSC.

normalize/normalize-audio

Una utilidad para ajustar el volumen de ficheros de audio. Si quieres normalizar el volumen de tus discos de vídeo, lo necesitas, pero no es de vital importancia.

ImageMagick

Un potente conjunto de herramientas en la línea de comando para dar formato, cambiar tamaños, convertir y dibujar en ficheros de imágenes. Se usa por makemenu, el componente generador de menús de tovid. Si quieres crear discos de vídeo con menús, lo necesitas. Muchas distribuciones Linux lo instalan por defecto.

SoX

El ImageMagick de procesamiento de audio, a su manera. Herramientas en la línea de comando para convertir, recortar, y filtrar ficheros de audio. Se usa por makemenu.

dvdauthor 0.6.0 o más nuevo

Herramientas para crear sistemas de archivo DVD. Si quieres hacer DVDs con tovid, lo necesitas.

vcdimager

Herramientas para crear sistemas de archivo (S)VCD. Si quieres hacer VCDs o SVCDs, lo necesitas.

transcode

Herramientas para procesar y convertir flujos de vídeo. Se usa por postproc (el componente de tovid para postprocesado) y dvrequant (el componente de tovid para encoger DVDs). Útil si quieres poder ajustar al detalle el tamaño de salida de tus videos, para que encaje la cantidad deseada de contenido en un DVD grabable.

libdvdcss

DVD decryption routines, used for playing encrypted DVDs on your computer. Needed by dvrequant too.

lsdvd

Práctica herramienta para ver los atributos de los discos DVD. También necesaria para dvrequant .

La mayor parte de estas herramientas están ampliamente disponibles en los principales repositorios de paquetes, lo más probable es que puedas obtenerlos todos usando el gestor de paquetes que incluya tu distribución Linux.

2. Instalación

2.1. Paquetes

  • tovid Gentoo ebuild. Mira Instalación de Ebuilds de terceras partes para ayuda en la instalación.

  • Debian .deb.

  • SuSE RPMs

2.2. Código fuente

  • tovid_0.21.tar.gz contiene todos los scripts de la línea de comando.

  • tovid-gui-0.21.tar.gz contiene la aplicación gráfica y sus iconos. Los scripts de la línea de comando han de estar instalados ante de poder usarse el interfaz gráfico.

Descárgate el paquete para la línea de comando y guárdalo en tu ordenador Supongamos que lo has guardado en ~/Downloads/.

Lo siguiente es abrir una ventana de terminal, (como XTerm, RXVT, o Konsole) y extraer el "tarball" al teclear los siguientes comandos:

$ cd ~/Downloads
$ tar -xzvf tovid_0.21.tar.gz

Los ficheros de instalación irán a un subdirectorio tovid_0.21 dentro del directorio en curso. Ahora, haced lo siguiente:

$ cd ~/Downloads/tovid_0.21
$ ./configure
$ su -c "make install"

Si decides que ya no quieres tovid, puedes ejecutar make uninstall desde el directorio original de instalación (~/Downloads/tovid_0.21).

Repite el procedimiento anterior para el paquete tovid-gui-0.21.tar.gz si quieres instalar el interfaz gráfico.

2.3. Versiones desde CVS

Puedes traerte una copia de la última versión de desarrollo de tovid por medio de CVS. Para obtener esta versión, necesitas tener CVS instalado.

Usar CVS es un poquillo diferente; en vez de simplemente descargarte tovid, estarás "desregistrando" una copia. (N. del T. Si "registrarse," es traducción de "check-in", entonces "desregistrarse" lo es de "checking out", forzada regla de tres de la traducción ;-) ) Usando el mismo directorio de ejemplo que anteriormente:

$ cd ~/Downloads
$ cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/tovid login
Logging in to :pserver:anonymous@cvs.sourceforge.net:2401/cvsroot/tovid
CVS password: (your email address)
$ cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/tovid co -P tovid
$ cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/tovid co -P gui
$ cd tovid
$ aclocal
$ autoconf
$ automake

Si estás instalando el interfaz gráfico, repite estos pasos en su directorio (aclocal, autoconf, y automake). Finalmente, ejecuta ./configure y su -c "make install", según se mostró anteriormente, en cada directorio respectivo para instalar tovid (y el interfaz gráfico) en tu sistema.

Nota

También hay disponibles a través de CVS tres módulos adicionales: docs contiene los fuentes DocBook (completos con ilustraciones) de la documentación que estás leyendo; test-tools es un conjunto de scripts diseñado para comprobar las opciones de codificación de mplayer y mpeg2enc (junto algunos otras propinillas); pytovid devendrá eventualmente en una completa re-implantación de tovid en lenguaje Python.

Si estás interesado en estas otras ramas de desarrollo, por favor coge los módulos CVS y contacta con los desarrolladores el canal #tovid de irc.freenode.net.
